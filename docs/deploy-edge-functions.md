# Edge Function `notify-appointment-telegram`: секрет ALLOWED_ORIGINS (продакшен)

## Зачем это новичку

Браузер при запросе к чужому адресу (в т.ч. к Edge Function на домене Supabase) присылает заголовок **`Origin`** — это «откуда открыт сайт», например `https://мой-сайт.ru`.

Функция **не** должна отвечать «разрешено всем подряд» (`*`), иначе любой сайт в интернете мог бы дергать её из JS. Поэтому в коде проверяется список **разрешённых origin**. Список задаётся **переменной окружения** **`ALLOWED_ORIGINS`**, которую в продакшене нужно положить в **секреты** проекта Supabase (они подставляются в функцию при выполнении в облаке).

**Важно:** подставь **ровно тот адрес**, по которому пользователи открывают твой фронт в браузере:

- С протоколом: `https://` или `http://` (в проде почти всегда `https://`).
- **Без** слэша в конце: `https://мой-сайт.ru`, а не `https://мой-сайт.ru/`.
- Несколько адресов (например, прод + превью) — **через запятую**, без пробелов:  
  `https://app.example.com,https://staging.example.com`

Локально секрет можно не задавать: если `ALLOWED_ORIGINS` пустой, в коде включены только `localhost` для разработки.

---

## Что сделать в продакшене (выбери способ)

### Способ A — через веб-интерфейс Supabase

1. Зайди в [Supabase Dashboard](https://supabase.com/dashboard) → свой проект.
2. Открой **Project Settings** (шестерёнка) → раздел **Edge Functions** → **Secrets** (или **Manage secrets** — название может чуть отличаться).
3. Добавь секрет:
   - **Name:** `ALLOWED_ORIGINS`
   - **Value:** твой URL фронта, например `https://мой-проект.vercel.app`
4. Сохрани. После смены секретов **задеплой функцию заново** (см. ниже), если деплой не подтянул секреты автоматически.

### Способ B — через Supabase CLI (из папки с проектом)

Установлен CLI и выполнен `supabase link` к облачному проекту:

```bash
npx supabase secrets set ALLOWED_ORIGINS="https://ТВОЙ-ДОМЕН.tld"
```

Несколько origin:

```bash
npx supabase secrets set ALLOWED_ORIGINS="https://app.example.com,https://www.example.com"
```

Кавычки нужны, если в значении есть символы, которые shell может исказить.

### Задеплоить функцию после изменений

Из корня репозитория:

```bash
npx supabase functions deploy notify-appointment-telegram
```

(Если секреты уже были и ты только обновил `ALLOWED_ORIGINS`, иногда достаточно сохранения секрета — но после первой настройки деплой гарантирует актуальный код.)

---

## Проверка

Открой продакшен-фронт, создай запись с уведомлением в Telegram. Если в консоли браузера ошибка CORS / `403 Forbidden origin` — значит `Origin` страницы не совпал со списком: исправь URL в `ALLOWED_ORIGINS` (другой поддомен, `www` vs без `www`, `http` vs `https`).

---

## Я не могу сделать это за тебя

Секреты живут **только** в твоём проекте Supabase. Репозиторий хранит лишь **имя** переменной и эту инструкцию; реальное значение с твоим доменом нужно ввести **ты** в Dashboard или через CLI.
