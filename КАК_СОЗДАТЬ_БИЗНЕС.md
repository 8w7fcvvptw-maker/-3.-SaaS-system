# Как создать бизнес в проекте (инструкция для новичка)

В этом проекте есть **два вида** «бизнесов». Оба создаются через базу данных Supabase.

---

## 1. Бизнес салона (businesses) — для Дашборда

Это ваш салон/барбершоп: с ним связаны клиенты, записи, услуги и сотрудники.  
**Без хотя бы одного бизнеса** не будут работать создание клиентов, сотрудников и записей.

### Шаг 1. Откройте Supabase

1. Зайдите на [supabase.com](https://supabase.com) и войдите в аккаунт.
2. Откройте свой проект.
3. В левом меню выберите **SQL Editor**.

### Шаг 2. Узнайте свой user_id

1. В Supabase откройте **Authentication** → **Users**.
2. Если есть пользователи — скопируйте **UUID** любого (длинная строка вида `a1b2c3d4-e5f6-7890-abcd-ef1234567890`).
3. Если пользователей нет — создайте: нажмите **Add user** → введите email и пароль → **Create user** → скопируйте UUID.

### Шаг 3. Создайте новый запрос

1. Нажмите **SQL Editor** → **New query**.
2. **Полностью удалите** текст в кавычках и вставьте **только** ваш UUID (без пробелов, без русских букв). Пример правильного формата: `'a1b2c3d4-e5f6-7890-abcd-ef1234567890'`

```sql
INSERT INTO businesses (user_id, name, description, address, phone, email, hours)
VALUES (
  'a1b2c3d4-e5f6-7890-abcd-ef1234567890',   -- замените на ваш UUID из Authentication → Users
  'Мой салон',
  'Описание вашего бизнеса',
  'г. Москва, ул. Примерная, д. 1',
  '+7 495 000-00-00',
  'info@mysalon.ru',
  'Пн–Вс: 09:00–21:00'
);
```

### Шаг 4. Запустите запрос

1. Нажмите кнопку **Run** (или `Ctrl+Enter`).
2. Если всё верно, появится сообщение вроде «Success. No rows returned» — значит запись добавлена.

### Шаг 5. Проверьте в приложении

1. Запустите проект (`npm run dev` в папке `frontend`).
2. Откройте раздел **Настройки** в меню слева.
3. Должны появиться поля профиля: название, адрес, телефон и т.д. — их можно редактировать.

## 2. Бизнесы для Admin-панели (admin_businesses)

Это список «клиентов» вашей SaaS-платформы — тех, кто пользуется системой. Они отображаются на странице **Admin → Бизнесы**.

### Шаг 1. Откройте SQL Editor в Supabase

(как в пункте 1 выше)

### Шаг 2. Вставьте и выполните запрос

```sql
INSERT INTO admin_businesses (name, plan, users, status)
VALUES 
  ('Салон «Красота»', 'Pro', 3, 'active'),
  ('Барбершоп №1', 'Free', 1, 'active'),
  ('Studio Nails', 'Enterprise', 5, 'active');
```

### Шаг 3. Проверьте

1. Откройте приложение и перейдите в **Admin** (если есть такой раздел).
2. Зайдите на страницу **Бизнесы** — там должны появиться добавленные записи.

## Важно

| Вопрос | Ответ |
|--------|-------|
| **Таблица `businesses` ещё не создана** | Сначала выполните SQL из `frontend/README.md` (раздел «3. Создать таблицы в Supabase») — там есть `CREATE TABLE businesses ...`. |
| **Ошибка «null value in column user_id»** | Таблица требует `user_id`. Скопируйте UUID пользователя из **Authentication → Users** и вставьте в запрос (см. Шаг 2 выше). |
| **Ошибка «column does not exist»** | Проверьте, какие колонки есть в таблице (Supabase → Table Editor → businesses). В примере выше используется user_id, name, description, address, phone, email, hours. |
| **Можно ли создавать бизнес через интерфейс?** | Сейчас — нет. Создание делается только через SQL в Supabase. Редактирование — в разделе **Настройки**. |
| **Сколько бизнесов можно создать?** | В таблице `businesses` — сколько угодно. Но приложение использует **первый** из списка. Для мульти-бизнеса нужны доработки. |

## 3. Тестовые данные (seed)

Готовый скрипт с демо-бизнесом, услугами, мастерами, клиентами и записями (со связями `business_id`, `service_id`, `staff_id`, `client_id`):

1. Выполните миграции связей из файла **`supabase-relations-migration.sql`** (в корне репозитория).
2. Supabase → **SQL Editor** → откройте **`supabase/seed.sql`**, вставьте содержимое → **Run**.
3. Нужен хотя бы один пользователь в **Authentication → Users** (для колонки `businesses.user_id`).

Повторный запуск скрипта не создаёт дубликаты строк с префиксом **SEED**.

## Краткая шпаргалка

1. Supabase → SQL Editor → New query.
2. Вставить SQL из этой инструкции.
3. Нажать **Run**.
4. Обновить страницу в браузере — данные подтянутся.
